# AI-Enabled Healthcare BI Copilot

A Streamlit portfolio project that combines healthcare business-rule documents, synthetic claims data, SQL analytics, and generative AI in one business intelligence workflow.

## What it does

- Displays healthcare claims KPIs and visual summaries from a SQLite database.
- Answers common BI questions using approved SQL patterns.
- Converts advanced natural-language questions into read-only SQLite queries.
- Uses retrieval-augmented generation (RAG) to answer questions about an uploaded healthcare or business-rules PDF.
- Combines document context with database analysis to generate policy-aware SQL.
- Automatically creates charts from query results.
- Requires explicit approval before an AI-generated query is executed.

## Technology

- Python and Streamlit
- SQLite and pandas
- OpenAI models through LangChain
- FAISS vector search
- PyPDF
- Matplotlib
- Synthetic healthcare data generated by Synthea

## Run locally

1. Clone the repository and enter its directory.
2. Create and activate a virtual environment:

   ```bash
   python -m venv .venv
   source .venv/bin/activate
   ```

   On Windows:

   ```powershell
   .venv\Scripts\activate
   ```

3. Install the dependencies:

   ```bash
   pip install -r requirements.txt
   ```

4. Copy `.env.example` to `.env` and add your OpenAI API key:

   ```text
   OPENAI_API_KEY=your_openai_api_key_here
   ```

5. Start the app:

   ```bash
   streamlit run app/main.py
   ```

## Data and privacy

The included database is a compact sample of synthetic Synthea healthcare
data intended for learning and demonstration. The larger raw CSV exports are
not included in the repository. Do not use real protected health information
or commit API keys. The local `.env` file is intentionally excluded from Git.
